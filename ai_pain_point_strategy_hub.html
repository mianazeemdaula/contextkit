The specific problem statement

Problem: Every AI power user has 3–10 "contexts" they use regularly — their company, their project, their writing style, their coding standards. These exist as scattered Notion pages, Google Docs, sticky notes, and long system prompts. Every new AI session means finding these, copying them in, and explaining everything again. There is no open-source, AI-agnostic way to manage, version, and instantly inject these contexts into any AI tool.
Solution: ContextKit Open source
An open-source, AI-agnostic context management system. Save your project context, personal profile, style guides, and prompts once. Inject them into any AI tool in one click. Works as a browser extension, CLI, and web app.

Core features

Context profiles
Save named contexts (Work, Personal, Project X) with full markdown support
One-click inject
Browser extension injects selected context into any AI chat with one click
Version history
Git-backed context versioning. See how your prompts evolved over time
Team sharing
Share context profiles across a team. Everyone starts with the same baseline
API + CLI
Pipe contexts into any workflow: ck inject my-project | claude
Public templates
Community-built context starters for common roles: developer, writer, marketer

Build phases

1
Week 1–2: Core CLI + local storage
  • TypeScript CLI with init, add, list, get, copy commands
  • Local JSON/markdown file storage in ~/.contextkit/
  • Publish to npm — npx contextkit works on first day
2
Week 3–4: Browser extension
  • Chrome/Firefox extension with popup showing saved contexts
  • One-click inject into ChatGPT, Claude, Gemini, Perplexity text fields
  • Reads local contexts via native messaging API

  • Next.js + Tailwind web UI for visual context management

Week 7–8: Cloud sync (SaaS layer)
  • Supabase backend for cloud storage + auth

Non-technical user strategy (the "live" version)

contextkit.app — the hosted no-code version Non-technical users land on a clean web app. They fill out a guided form: "Tell me about your role", "What project are you working on?", "What's your preferred tone?" — the app auto-generates their context profile. One click copies it into any AI tool. No GitHub, no CLI, no setup. This is the product they pay for.

Acquisition loop: Technical users find it on GitHub → they share the web app with non-technical teammates → teammates subscribe. This is exactly how Notion, Linear, and Vercel grew.
